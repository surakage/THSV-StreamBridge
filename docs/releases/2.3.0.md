# THSV StreamBridge 2.3.0

Version 2.3.0 expands the optional creator-automation suite while keeping every add-on independently installable, permission-scoped, and removable.

## New add-on downloads

- **Scene Actions 1.0.0** — maps exact OBS Studio, Streamlabs Desktop, and Meld Studio scene changes to creator-approved Streamer.bot actions through the existing bridge connection.
- **First Five 1.0.0** — manages five sequential Twitch Channel Points reward positions, prevents repeat claims, refunds invalid redemptions, and keeps a private monthly leaderboard.
- **Fan Crown 1.0.0** — manages a rotating Twitch “No. 1 Fan” reward with bounded pricing, serialized claims, refunds, monthly rankings, manual maintenance actions, and an optional overlay card.
- **Raid Scout 1.0.0** — finds a bounded Twitch raid candidate from preferred, followed-live, or same-category channels, with safety filters, repeat protection, private suggestions, and creator confirmation by default.

## Improvements

- Automated Shoutouts 1.1.0 adds an optional Twitch profile card or bounded random-clip popup while keeping YouTube, Kick, and TikTok shoutouts chat-only.
- Add-on settings use shorter collapsible panels, switches, selectors, and conditional fields so advanced controls stay hidden until needed.
- Raid Scout separates discovery, audience, exclusions, messages, overlay content, styling, and privacy into guided setup sections.
- Hosted add-on overlays use shorter creator-facing URLs while preserving existing module-ID routes.
- Scene changes and add-on relay results continue to share StreamBridge’s existing authenticated Streamer.bot WebSocket instead of opening extra connections.

## Installation

Download the core Windows ZIP and its adjacent checksum. Each optional add-on is a separate ZIP containing:

- the wizard-installable `.thsv-addon`;
- its matching Streamer.bot `.sb` import;
- installation notes; and
- an adjacent SHA-256 checksum.

Install core first. Then install only the add-ons you want, approve their exact Streamer.bot actions in the wizard, and run manual or simulated acceptance checks before using live rewards, raids, clips, shoutouts, or scene automation.
