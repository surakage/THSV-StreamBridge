# THSV StreamBridge 2.4.0

Version `2.4.0` is the unified stable baseline for the bridge, every first-party add-on, and every Streamer.bot package.

## Highlights

- All 12 first-party add-ons and all 24 Streamer.bot imports now use the same visible `2.4.0` release version.
- The normalized event and add-on API contract remains `2.0.0-preview.1`; it is an interface compatibility version, not an outdated installer.
- Quote Vault adds one moderated, bounded, local quote library shared by Twitch, YouTube, Kick, and TikTok.
- Discord Chat Archive adds disabled-by-default, filtered public-chat forwarding to a creator-owned Discord webhook without keeping a local transcript or webhook secret in StreamBridge.
- Every add-on uses a short introduction and collapsible setup sections. Random Clip Player now has guided setup, Subathon visuals are split into layout and color groups, and all settings pages have Expand all and Collapse all controls.
- Core and add-on update checks provide direct trusted GitHub asset links. Add-on results also show the exact archive name and published SHA-256.

## Safe updates

StreamBridge never silently executes an update. The wizard checks the official release, links the exact package, and leaves installation under creator control.

For core:

1. Download `THSV-StreamBridge-2.4.0.zip` and its adjacent checksum.
2. Verify the archive.
3. Close StreamBridge and run the included installer.
4. The installer preserves creator data, verifies activation health, and rolls back a failed activation.

For an add-on:

1. Open **Wizard > Add-ons > Check updates**.
2. Download the exact verified add-on ZIP shown for that add-on.
3. Install its `.thsv-addon` in the wizard and import its matching `2.4.0` `.sb` file.
4. Review permissions and action grants, then restart StreamBridge when requested.

## Compatibility and safety

- Streamer.bot `1.0.5-alpha.33` is recommended; package minimums remain at the last verified compatible floor.
- High-impact operations without verified provider-stable IDs remain blocked.
- Add-on files are hash-verified and permission-reviewed independently.
- Executable add-ons share the StreamBridge Windows account and are not an operating-system sandbox.
