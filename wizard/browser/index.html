Template tokens — what each one means
{actor}
Who triggered the alert. Falls back to "The community" when no user is attached (like-milestone alerts).
{alertType}
The alert's own category, e.g. "gift subscription" — useful when one template covers several alert types.
{platform}
Which platform the event came from: twitch, youtube, kick, or tiktok.
{amount}
The monetary amount. Only set for cheers with bits, and Super Chat / Super Sticker.
{currency}
The ISO currency code that goes with {amount}.
{quantity}
A count — gifted subscriptions, cheer bits, or gifted months — depending on the alert type.
{itemName}
The name of the gifted item, when the platform provides one (e.g. a TikTok gift's name).
{tier}
The subscription tier, when the platform reports one.
{message}
Any accompanying text — a cheer message, a resub message, and so on.
{metric}
Which metric a like-milestone alert is about (currently only "likes").
{value}
The metric's numeric value for a like-milestone alert.

Quantity aggregation is available only for gifts and gift subscriptions.

Channel rewards

Twitch reward controls compile in Streamer.bot v1.0.5-alpha.31, but live state-changing behavior has not been exercised. Streamer.bot exposes no documented reward-listing request, so copy the exact custom reward GUID from Twitch or Streamer.bot. Each submit is an immediate live operation after confirmation; it is not part of the wizard's staged configuration save. Kick redemption intake is supported, but Kick mutation controls are disabled because their official contract is not documented.

Add-ons

Install only .thsv-addon packages you trust. Declarative add-ons contain settings and metadata only; executable add-ons can run code with the permissions shown here. Add-ons share the existing bridge process, Streamer.bot connection, and core-hosted overlay transport.
